# 스마트 창고 출고 지연 예측 경진대회, 연습

창고의 실시간 운영 상태 데이터를 기반으로 **향후 30분간의 평균 출고 지연 시간(분)**을 예측하는 머신러닝 회귀 모델입니다.

> 검증 MAE **9.98 → 9.19** · 리더보드 **11.62 → 10.87** (약 25% 개선)

---

## 문제 정의

물류 창고에서 출고 지연은 배송 지연, 고객 만족도 하락, 운영 비용 증가로 직결되는 핵심 운영 리스크임. 지연이 발생할 시점을 미리 예측할 수 있다면, 작업자·로봇 자원을 선제적으로 재배치해 지연을 완화할 수 있음.

- **문제 유형**: 회귀 (Regression)
- **예측 대상**: `avg_delay_minutes_next_30m` (향후 30분 평균 출고 지연 시간)
- **평가 지표**: MAE (Mean Absolute Error)

---

## 데이터

| 파일 | 규모 | 설명 |
|------|------|------|
| train.csv | 250,000 × 94 | 학습 데이터 (타깃 포함) |
| test.csv | 50,000 × 93 | 평가 데이터 |
| layout_info.csv | 300 × 15 | 창고 레이아웃 특성 |

### EDA로 발견한 핵심 특징

- **훈련/평가 데이터의 창고·시나리오가 0% 겹침** → "학습에서 본 적 없는 새 창고"에서도 예측해야 하는 일반화 문제
- **타깃이 오른쪽으로 크게 치우침** (중앙값 9분, 평균 19분, 최대 715분)
- 90개 피처 중 86개에 약 12%씩 결측 (분석 결과 무작위 결측)

> 데이터는 대회 규정상 재배포하지 않음.

## 접근 방법

### 1. 식별자 제거
`ID`, `layout_id`, `scenario_id` 제거.
**근거**: train/test 간 창고·시나리오 겹침이 0%임을 직접 검증. 학습에서 본 식별자가 평가에 없으므로 피처로 쓰면 과적합만 유발.

```python
train_ids = set(train['layout_id'])
test_ids = set(test['layout_id'])
print(len(train_ids & test_ids))   # 0 → 겹치지 않음
```

### 2. 타깃 로그 변환
치우친 타깃에 `log1p` 변환 후 예측값을 `expm1`로 역변환.
**근거**: 소수의 극단적 지연값에 모델이 휘둘리지 않고 전 구간을 고르게 학습하도록 유도.

### 3. 검증 전략 재설계 — GroupKFold (핵심 전환점)

일반 KFold 검증은 MAE 7.87로 낙관적이었으나 실제 리더보드는 11.62로 큰 괴리가 있었습니다. 원인은 일반 KFold가 **같은 창고를 학습·검증에 함께 사용**하여 "이미 본 창고"를 평가한 것이었습니다.

**GroupKFold(layout_id 기준)**로 창고 단위 분리 검증을 도입하여 "새 창고"로 평가 → 실제 환경을 정확히 재현.

| 검증 방식 | MAE | 의미 |
|-----------|-----|------|
| 일반 KFold | 7.87 | 낙관적 (같은 창고 혼입) |
| GroupKFold | 9.69 | 실제 성능 반영 |

### 4. 과적합 진단 및 규제
검증-실제 격차를 과적합으로 판단. Optuna로 GroupKFold 기준 재튜닝하며 규제(`reg_alpha`, `reg_lambda`) 탐색 → 낮은 학습률 + 강한 L2 규제가 선택됨 (과적합이 실제로 존재했음을 입증).

### 5. 도메인 피처 엔지니어링 (최대 성과)

**창고 ID는 겹치지 않아 쓸 수 없지만, 창고의 물리적 특성(통로 폭, 면적, 로봇 수 등)은 연속값이라 새 창고에도 일반화된다**는 통찰로 `layout_info.csv`를 병합.

나아가 운영 데이터와 창고 특성을 결합한 파생 피처를 설계:

```python
# 포장 처리 능력 대비 주문 부하 → 변수 중요도 2위
df['order_per_packstation'] = df['order_inflow_15m'] / (df['pack_station_count'] + 1)
```

### 6. 앙상블
LightGBM + XGBoost + RandomForest를 각각 GroupKFold 기준으로 튜닝 후 예측 평균.

## 결과

| 단계 | 검증 MAE | 리더보드 |
|------|:--------:|:--------:|
| 베이스라인 (평균값 예측) | 16.77 | - |
| LightGBM 기본 | 9.98 | - |
| + 타깃 로그 변환 | 9.21 | - |
| + GroupKFold 검증 & 규제 튜닝 | 9.43 | 11.04 / 11.28 |

### 출고 지연 주요 요인 (변수 중요도)

- **작업 처리 능력**: 포장 가동률, 평균 이동 거리, 컨베이어 속도
- **주문 특성**: SKU 집중도, 주문당 품목 수, 긴급주문 비율
- **로봇/설비 상태**: 배터리 평균, 활성 로봇 수, 충전 효율
- **포장 부하 지표**: 주문 유입 ÷ 포장대 수 (직접 설계, 중요도 2위)

## 배운 점

- **올바른 검증이 절반이다.** 검증 점수가 실제와 괴리되면 모델이 아니라 검증 설계를 의심해야 한다.
- **큰 향상은 기법이 아니라 데이터 이해에서 온다.** 모델 튜닝·앙상블로는 9.43이 한계였으나, 창고 특성 활용과 도메인 피처 설계로 9.19까지 돌파했다.
- **피처는 만들고 → 검증하고 → 효과 없으면 버리는 반복이다.** 무작정 추가한다고 좋아지지 않는다.

### 향후 방향
예측된 지연 정보를 활용해 작업자·로봇 자원 배분과 출고 우선순위를 **최적화**하는 의사결정 모델로 확장 (predict-then-optimize).
